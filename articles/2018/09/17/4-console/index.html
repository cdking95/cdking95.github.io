命令

Seeruo的可用命令很少，你可以根据你的需要，自己开发插件扩展命令；
如果忘记命令，可以执行help命令，获取‘命令使用帮助’。

php seeruo --help

Seeruo的所有命令都需要在项目所在根目录下执行，即seeruo所在目录

1.创建文件

php seeruo create test //不需要加文件后缀

文件创建完成之后在Source里面可以看到一个名为'test.md'的文件，你可以选择你自己喜欢的编辑器去撰写文章。

2.构建静态网站

构建网站有两种模式，第一种：这种模式是单次构建，不管之前构建的是否正确，本次都会执行一次构建操作

php seeruo build

第二种：这种模式会实时监测(源文件、配置文件、模版文件)文件的变动，如果有任意一个文件有变动，都会重新执行一次构建操作

php seeruo build -s

生成的静态文件会存放在Public文件夹；

3.本地预览

网站构建完成之后，需要预览网站的展示效果，可以通过php只带的本地调试服务器预览，也可以通过自带的简单网页服务器预览：
使用php自带的服务器：

php -S localhost:9001 -t Public/

使用Seeruo自动的服务器：

php seeruo server

两种方式都可以。

4.发布到生产环境

最后一步，网站构建完成之后需要发布到服务器，供大家浏览。
发布的方式现在有两种，一种是ssh发布，一种是git发布，需要根据不同的应用场景，使用不同的方式；

第一种: ssh
这种方式，是针对类似‘阿里云’服务器的方式，任何可以通过ssh链接的服务器，都可以通过这种方式发布；
发布之前，请先确认已经配置好了ssh所需要的配置

php seeruo ssh push

第二种: git
这种方式的相对于ssh要麻烦一点点，需要先执行一次初始化操作, 才能执行发布操作；

php seeruo git init
php seeruo git push
